# 理论知识课件生成配置

## 课件生成要求

### 基本要求
当用户要求生成课件时，必须遵循以下要求：

1. **搜索网络获取资料**
   - 使用 web_search 工具搜索相关主题的最新资料
   - 查找权威学术机构、知名公司的相关材料
   - 收集相关的文章、视频资源

2. **生成详细教学材料**
   - 为指定部分生成详细的课件内容
   - 采用教授教学生的方式详细解释概念
   - 知识点每一部分都需要仔细拆解，就像大学教授浅显易懂教学生
   - 添加实际生活案例和应用场景
   - 然后才是结合代码

3. **基于 app.py 解释代码**
   - 详细分析 app.py 中与该部分相关的代码
   - 解释代码的工作原理和实现细节
   - 说明代码在实际项目中的作用

4. **添加权威资源**
   - 引用权威学术机构的材料
   - 添加知名公司的相关资源
   - 提供相关的文章和视频链接

5. **创建闪卡检测系统**
   - 在每个知识点讲解后立即添加闪卡
   - 闪卡用于测试刚刚讲完的知识点
   - 闪卡背面答案通过点击显示，不直接显示
   - 提供即时反馈和复习机制
   - 每个主题包含10个不同类型的闪卡：
     - 选择题：测试概念理解
     - 简答题：考察知识点掌握
     - 填空题：检验关键术语记忆
     - 写代码题：实践编程能力
     - 分析代码错误：培养调试能力
     - 逻辑实现题：训练算法思维
     - 阅读代码题：理解代码功能和解决的问题

6. **文件命名规范**
   - 新建文件放在"理论知识"目录下
   - 文件名要清楚表明这是该文件的详细课件
   - 建议格式：topics内容_lesson.md
