## 2. Part A — Saving for a House

You have just graduated from MIT and have a job! You move to the Bay Area and decide to start saving for a home. Your goal is to find the number of months it takes to save up for a down payment.

### 2.1 User Inputs
Ask the user to enter the following variables and cast them as floats, **in this order**:

- `yearly_salary` — the starting yearly salary
- `portion_saved` — the portion of salary to be saved, in decimal form (e.g. `0.1` for 10%)
- `cost_of_dream_home` — the cost of your dream home

### 2.2 Program Variables
The following variables are fixed:

- `portion_down_payment = 0.25` — 25% of total home cost
- `amount_saved = 0` — starts at $0
- `r = 0.05` — 5% annual rate of return

### 2.3 Monthly Update Formula
At the end of each month, your savings increase by:

```python
amount_saved += (yearly_salary / 12) * portion_saved
amount_saved += amount_saved * (r / 12)
```

> **Note:** The investment return is calculated on the amount saved at the **start** of each month.

### 2.4 Output
Your program should store and print the number of months required:

```python
months = <calculated value>
print("Number of months:", months)
```

| Case 3 | yearly_salary: `350000` / portion_saved: `0.30` / cost: `10000000` | `Number of months: 189` |

## 4. Part C — Choosing an Interest Rate

In Part C, we flip the question: given a fixed **initial deposit**, find the **lowest rate of return** `r` that allows you to save enough for a down payment in exactly **3 years**, using bisection search.

### 4.1 Fixed Assumptions
- House cost: `$800,000`
- Down payment: `25%` → target = `$200,000`
- Time horizon: `36 months`
- Acceptable range: within `$100` of the target (exclusive)

### 4.2 User Input
- `initial_deposit` — cast as float

### 4.3 Compound Interest Formula

```
amount_saved = initial_deposit × (1 + r/12) ^ months
```

### 4.4 Bisection Search Logic
- Search `r` between `0.0` and `1.0` (0% to 100%)
- Bisect until `amount_saved` is within `$100` of the down payment
- Store the number of bisection steps in `steps`

### 4.5 Special Cases
- If `initial_deposit >= down_payment - $100` → `r = 0.0`
- If no valid `r` exists within 0%–100% → `r = None` (not the string `"None"`)

### 4.6 Notes
- `None` is Python's null value — different from the string `"None"`
- If your program runs forever, check your stopping condition for an infinite loop
- Your `steps` count may vary slightly from the examples — the tester allows for this

### 4.7 Test Cases

| Test Case | Input | Expected Output |
|-----------|-------|-----------------|
| Case 1 | `initial_deposit: 65000` | `Best savings rate: 0.380615234375` / `Steps: 12` |
| Case 2 | `initial_deposit: 150000` | `Best savings rate: 0.09619140625` / `Steps: 11` |
| Case 3 | `initial_deposit: 1000` | `Best savings rate: None` / `Steps: 0` |
